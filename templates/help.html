A server for serving and rendering files in localhost.
This is a web-based file explorer and its root path is your ~/Pylocalhost directory.
It is useful for serving HTML,Ajax,PHP and every thing that needs a webserver.

Logging in:

Installer will give you a 5 digit password for loggin in. If you don't login you wont be able to use all Pylocalhost options; that means PyLocalHost will be read-only for you or anyone else who did not log in.Remember anyone can login to your PyLocalHost if he know your password and also can use your PyLocalHost in read-only mode even if he does not have your password. You can prevent this by disallowing requests from unknown hosts in settings.
You can go here => in order to login.

If you have forgotted your password run this to see it again:

$ sudo pylh password

CLI Usage:

$ sudo pylh <command>

Commands:
  • start --> Start Pylocalhost
  • stop --> Stop Pylocalhost
  • restart --> Reload config files & restart Pylocalhost
  • enable --> Enable run as start up for Pylocalhost
  • disable --> Disable run as start up for Pylocalhost
  • password --> Print your password
  • help --> Show help
  • ? --> Show help

Settings guid

Allow requests from unknown hosts:
Allow this option if you want to share your Pylocalhost over network
If you disallow this option Pylocalhost will be available only for logged-in users so other people cannot access to your pylocalhost.
NOTE: This option is allowed by default
WARNING: If you allow this option your Pylocalhost will be shared over network be sure and aware about what are you doing.

Show hidden files:
Select Show option if you want Pylocalhost to show Unix hidden files (starting with . )
NOTE: This option is disabled by default

Default editor theme:
You can choose the default theme for PyLH editor (for editing & creating files)

Toolbar guid

You can see a toolbar above the files and folders that has some items:

Returns you back to the home page of localhost which shows files in your ~/Pylocalhost

Keeps you to the parent directory

Opens the current directory using your systems default file browser/manager. If it fails nothing will happen

Serves current directory using nginx so it will show you the index file or forbidden page

Starts a Jupyter notebook on current directory.(If you had accepted Jupyter installation when you was installing PyLocalHost)

If it fails nothing will happen

Stop all running Jupyter notebooks.If it fails nothing will happen(It may not work)

Create a new file in current directory

Create a new folder in current directory

Address Shows the path of current directory

Right click menu guid

By right clicking on a file,folder or ... you can see a list of actions that you can do with that file or ...:
NOTE: Items with * just can be used by logged-in users

View source => Shows the raw file as a plain text
Download => Download the item
Delete * => Deletes item
Rename * => Renames item
Edit * => Opens editor in order to edit the file
Open with OS * => Opens the item using suggested software of your operating system. If it fails nothing will happen
Run script * => Runs the file (Just Python & JavaScript files) and shows output

NOTE : Pylocalhost will return you a JSON api including info of contents of a directory if you add ?api=ture at the end of directory urls.
For example:
http://pylh/folder1/folder2/ (Returns the GUI file manager) --> http://pylh/folder1/folder2/?api=true (Returns JSON)
